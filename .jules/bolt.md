## 2025-07-06 - Array.prototype.reduce overhead
**Learning:** In the Bun/V8 runtime, using `Array.prototype.reduce` for string concatenation (e.g. `arr.reduce((acc, x) => acc + x, "")`) is notoriously slow due to repeated string allocations causing O(N^2) time complexity in practice. Furthermore, executing multiple sequential `reduce` passes over the same array to calculate distinct aggregates introduces unnecessary callback overhead and increases complexity from O(N) to O(k*N).
**Action:** Replace `reduce` string concatenations with standard `for` loops or `.map().join()`, which handle buffer allocations far more efficiently. Combine multiple mapping/reducing passes over the same data into a single `for` loop to avoid overhead.
