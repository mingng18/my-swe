## 2023-10-27 - Sequential Awaits with Dependencies
**Learning:** Static analysis tools sometimes incorrectly suggest optimizing sequential `await`s into a `Promise.all` even when the second asynchronous operation depends on the resolved value of the first. In this case, `harness.run` depends on the `harness` instance returned by `await getAgentHarness()`.
**Action:** Always verify data dependencies between sequential awaits before attempting to parallelize them. If a dependency exists, document the constraint with an inline comment instead of making functional or purely syntactic changes that don't yield actual performance benefits.
